2. **Description:** Add a short description of the Template.
3. **Document Link:** In the Document Link field enter your document code. The system will access the document by the code number then apply the template. _**This field is designed for enterprise customers and is dependent on our API to connect to custom documents.**_ You can skip this step if you you do not have an eSign enterprise setup.
4. Scroll down to the **Documents** section.
5. Drag and drop your file into the upload area, or click **Upload** to select a file from your computer.
6. Upload your document. There are three options available for adding a document to a Template:
1. Drag and drop your documents to the documents section.
2. Upload – From your computer or local drive, navigate to the document(s) then select them.
3. Get from Cloud – Select from OneDrive, Dropbox or box, then navigate to the documents and select them.
7. The uploaded file will appear in the list below with its name, size, and the number of pages.
1. Scroll down to the Signer Roles section.

4. Add the Signer Roles: **Tip:** **Add the maximum number of Signer Roles that are appropriate for the document. It’s easier to remove unnecessary tags when creating a signing session. Click the +Add Role button and in the pop-up modal select the signer roles, and click the Select button.**

## Procedure 16: 12-2-2024 - Upcoming Changes to the OneKey MLS Data Feed

**Steps**:

1. **Listings with ListAOR as LIBOR:** The ListingId will now include a prefix "L".
2. **Listings with ListAOR as HGAR:** For entries where the OriginalEntryTimestamp is after 2024-11-12, the ListingId will no longer include the "H" prefix.
3. **New Listings Post-Migration:** Listing IDs will be numeric and will not carry any prefixes. This will be true of both HGAR and LIBOR.
4. **MemberMlsId Changes:** Some MemberMlsIds will be updated. Legacy IDs can be found under AltMemberMlsId.
