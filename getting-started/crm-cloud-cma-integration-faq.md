
# CRM Cloud CMA Integration FAQ

## **RDESK / CLOUD CMA INTEGRATION**

**Frequently Asked Questions (FAQs)**

**Why am I unable to see Cloud CMA section in my rDesk account?**

This feature is optional and will be available only for clients who has requested this feature to be turned on.

**Do I need to have an account with Cloud CMA to use this feature?**

It is mandatory that the user have an account with Cloud CMA and that the email address associated with rDesk account and Cloud CMA account should be identical.

**Why am I unable to use the Cloud CMA section in rDesk though I have an account with Cloud CMA?**

The email address associated with rDesk account and Cloud CMA should be identical. This feature will not work if the email addresses are different.

**What should I do if my email address for rDesk account doesn't match with the email address for Cloud CMA?** 

The email address tied to Cloud CMA account should be changed to match email address associated with rDesk CRM account.

**How do I change my email address in Cloud CMA account?**

1.  Login to your Cloud CMA account.
2.  Click on “Settings” next to the “Sign out” button
3.  In the “E-mail” box change the email address to match the email address associated with rDesk CRM account
4.  Click on “Update” button to save the changes

**How do I view my reports when I login to Cloud CMA section for the first time?**

Click on IMPORT button to import all the unpublished/published reports from Cloud CMA website.

**How do I assign a contact to a report imported from Cloud CMA?**

After importing the report from Cloud CMA, in the Contact field click on "Assign to Contact" to assign the report to a particular contact. 

**How do I view a published report after I import a report from Cloud CMA website?**

Click on the screen icon to view the report.

**How do I edit an unpublished/published report after I import a report from Cloud CMA website?**

Click on the pencil icon to edit the report.

**Can I sort my imported reports?**

Yes, the reports can be sorted by various fields in the Cloud CMA section.
