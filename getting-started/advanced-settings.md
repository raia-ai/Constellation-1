
# Advanced Settings | Constellation1 Customer Hub

Advanced Settings allows users to customize the application to suit your business requirements as well as letter templates and custom fields. 

For more information about Templates and their customization, please refer to our Knowledge Base; [Customizing Templates](https://constellation1.na3.teamsupport.com/knowledgeBase/16408636)

For more information about Custom Fields, please refer to our Knowledge Base; [Agent Custom Fields](https://constellation1.na3.teamsupport.com/knowledgeBase/23449147)

**System card**

**Accounting System** will display one of the options below; 

*   QuickBooks® Desktop
*   QuickBooks® Online 
*   No Accounting Integration
*   Xero

**Download Sync Manager**

*   This may be downloaded if you are integrating with QuickBooks® Desktop during your Onboarding with a member of our Team

**Enable Tax**

*   Toggle ON, if you are required to pay tax on commission

![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/7e348367-a277-4909-ba0d-63d4096980b4)

*   Select \+ Add Tax
*   Tax Name; enter the name of the tax such as HST, GST, etc. 
*   Tax Number; enter the general ledger number 
*   Tax Rate; enter the rate of tax up to 4 decimal places
*   Toggle ON, Enable Tax Automation if applicable
*   If Enable Tax Automation is enabled, select the Sync icon to retrieve the applicable Accounting Tax Code and Accounting Tax Exempt Code from the drop down
*   Select **Save** 

![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/f9df67ec-125e-4b0b-bb90-0b4a1eb13a50)

To edit or delete tax, select the ellipsis in the Action column.

**Transaction Settings card**

 ![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/5f1b39bc-7c99-4e20-b5a2-b96b1e6c57dc)

**Enable Transaction Number Auto Generation**

*   If engaged (toggled ON), allows users to enter a number that will attach to all transactions and sequentially increase by 1
*   Users can configure a Transaction Number Prefix such as T2024 to identify which transactions were entered in 2024 as an example

**Enable Transaction Wizard**

*   If engaged (toggled ON), will guide users through the steps to add a transaction; please select one of our Knowledge Base articles for further information; [Creating Transactions via Transaction Wizard](https://constellation1.na3.teamsupport.com/knowledgeBase/15356015) or [Adding a Transaction Via the Transaction Wizard (Video)](https://constellation1.na3.teamsupport.com/knowledgeBase/23826668)
*   If not engaged, users will manually add a transaction; please select one of our Knowledge Base articles for further information; [Manually Adding a Transaction](https://constellation1.na3.teamsupport.com/knowledgeBase/21273607) or [Manually adding a Transaction (Video)](https://constellation1.na3.teamsupport.com/knowledgeBase/22105084)

**Enable Commission Wizard**

*   If engaged (toggled ON), will guide users through the steps to add a commission plan; please select one of our Knowledge Base articles for further information; [Creating Commission Plans Using Commission Wizard](https://constellation1.na3.teamsupport.com/knowledgeBase/14674201) or  [Creating Commission Plans (Video)](https://constellation1.na3.teamsupport.com/knowledgeBase/22063920)
*   If not engaged, users will manually add a commission plan

**Enable Conditions Tracking**

*   If engaged (toggled ON), allows users to configure applicable conditions for their business requirements. Please select one of our Knowledge Base articles for further information; [Conditions Tracking](https://constellation1.na3.teamsupport.com/knowledgeBase/20920341) or [Conditions Tracking (Video)](https://constellation1.na3.teamsupport.com/knowledgeBase/22918604)
*   If not engaged, users will not be able to attach conditions to transactions 

**Bypass A/R Sync on Transactions Closing**

*   By default this is toggled OFF if your brokerage does not integrate with an accounting application 

**Exclude Deductions from Capped Amount Calculation**

*   If engaged (toggled ON), the application will not include any deductions attached to the transaction to be included in the agents commission plan calculations
    
*   If not engaged, the application will include any deductions attached to the transaction be included in the agents commission plan calculations
    

**Enable Auto-Save for Transaction Field Changes**

When enabled, this feature will automatically save any changes made to transactions.   
The saved data will cover the following fields:

*   Transaction Details
    *   All fields on the Property Card.
    *   All fields on the Additional Details Card, including custom fields.
    *   The four fields at the top of the Deposit Card. (Held By, Required Deposit Amount, Balance Due Date, Deposit Surplus Payee)
    *   All fields on the Franchise Card. 
*   Commissions Tab
    *   All Additional Detail fields on the Commission Detail Card.
    *   Buying and Selling side broker’s name. 

**Remember last used Disbursed By and Disbursement Form**

*   If engaged (toggled ON), all transactions added or imported will automatically inherit the last Disbursed By contact and Disbursement Form.
    
*   If not engaged, users will select/add both the Disbursed By contact and Disbursement Form manually if applicable

![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/aae069f7-844a-4a83-b270-582122ae1e4f)

**Default City**

*   If engaged (toggled ON), all transactions added will automatically inherit the default city selected in this field
    
*   If you import from a TMS such as Skyslope or Dotloop, the city entered in TMS will be inherited and **NOT** the default city selected

**Field Settings card**

![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/149835ea-80e2-4993-9e52-a822becf9e41)

**Field settings** allows users to custom their Transaction Types, Property Types, Lead Sources and Conditions (if Enable Conditions Tracking is engaged)

*   To **Add** a label, select the ![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/f4d67731-85a6-4b42-a9cb-3b6451df88a3)  icon and enter the label you require
*   To **Edit**, hover over an existing label, select the ![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/004cf30e-d313-4ddf-8fa9-87dcd9c7e539) icon and make your changes
*   To **Delete** a label, hover an existing label, select the ![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/90bc5b75-b49d-407c-b81c-e7e1c8eef57c) icon to delete the label
*   To by pass the automatic alphabetical sort order, select the ![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/6f2b45a2-099a-4072-bd71-e2bb86a23cde)icon which will then move this field to the top of the list within the transaction  

**SSO Settings card**

SSO or Single Sign On settings is used for customers who have their own SSO account with an outside provider. ie. OKTA, Google etc.

If you have recently acquired this functionality and would like to have it added to your account, please email [commissions@constellation1.com](mailto:commissions@constellation1.com) and a member of our Team will be happy to assist.
