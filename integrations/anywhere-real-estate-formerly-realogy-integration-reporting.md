# Anywhere Real Estate℠ (formerly Realogy) Integration > Reporting

This article details the steps required to transmit your Transactions, Agents, Teams and Keypersons to Anywhere Real Estate℠.

On the left navigation bar select **Reports** / Franchise Reporting tab / Mappings card

Please ensure you map all Transaction Types as needed. The label on the left of each drop down are the types you created, select the Anywhere Real Estate℠ required type from the drop down accordingly and select Save.

![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/26a9e65a-8818-450d-ac2c-40f1912294ef)

Ensure you map all Property Types as needed. The label on the left of each drop down are the types you created, select the Anywhere Real Estate℠  required type from the drop down accordingly and select Save.

![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/49a34721-a88c-45f6-9da2-170e1042312c)

Ensure you map all Lead Sources as needed. The label on the left of each drop down are the sources you created, select the Anywhere Real Estate℠ required source from the drop down accordingly and select Save.

![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/384cd71d-391e-444c-9a75-0a98e0d41a15)

To customize the Transaction Type, Property Type or Lead Source, please review [Advanced Settings](https://constellation1.na3.teamsupport.com/knowledgeBase/20368643) or [Advanced Settings ( Video)](https://constellation1.na3.teamsupport.com/knowledgeBase/22297060)

On the Users card, you will select the applicable Person Type; Agent, Team or KeyPerson. A keyperson can be defined as all users except an Agent; for example the Master, Office Administrator or Management user.

The Modified Since date will include all Person Types edited since date selected. If you are required to Include Inactive Users, the toggle should be ON.

Select Process

![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/a6bb0b01-6ed4-425a-b0c6-7591434690be)

Select the down arrow to see the results.  Any failures will provide you with the details of the failure so you can fix and re-process as needed.

![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/e2581d50-7e7d-4fd7-b2ba-f8ddb33c6c26)

In the Transactions card, you will select the applicable status(es). The Modified Since date will include all transactions edited since date selected. Last Sync date will appear at top right. 

When you expand Transactions, you will see the transactions that meet the criteria selected. Select Process.

![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/ebf171e4-3daf-4486-8ade-7adfa8d79444)

On the Results card select the Office(s), Integration Entities (Agent, Team, KeyPerson and Transaction) and Result Type (Success or Failed), then Retrieve to see the results.

![](https://constellation1.na3.teamsupport.com/api/attachments/action/1/a9eb68cf-6080-4518-b956-1b2fbc6277b3)
