# Constellation1 eSign July 2024 Release Notes

Production Release: July 23, 2024

## Release Summary  

**Signer Tags User Interface Update:** Text is now bottom justified and added drag icon.  
**User Branding:** Added user’s initials to the header.  
**Bug Fixes**

## Release Details  

Signer Tags User Interface Updates  
The user interface for signer tags has been updated to bottom-justify the text within the tag boxes. This change from the previous top-justified text layout improves the visual display of signatures, dates, and other information entered into the tags on documents.

 _![](https://app.na3.teamsupport.com/Wiki/WikiDocs/2472652/images/KB-ReleaseNotes/eSign/2024/07%20July/2024-July-eSign-UserIconbefore.jpg)_  
_Before_

  _![](https://app.na3.teamsupport.com/Wiki/WikiDocs/2472652/images/KB-ReleaseNotes/eSign/2024/07%20July/2024-July-eSign-UserIconafter.jpg)_  
_After_

A hand icon has also been added to all markups and tags, on the desktop and mobile versions, to visually indicate that the tags can be dragged and repositioned elsewhere on the document. 

![](https://app.na3.teamsupport.com/Wiki/WikiDocs/2472652/images/KB-ReleaseNotes/eSign/2024/07%20July/2024-July-eSign-DragIcon2.jpg)

  
 Added User Branding to Header  
The user interface has been updated to display the logged-in user's initials in the header, replacing the "person" icon in the top right corner. Users can click on their initials to access their account profile and quick start guide.

  
 ![](https://app.na3.teamsupport.com/Wiki/WikiDocs/2472652/images/KB-ReleaseNotes/eSign/2024/07%20July/2024-July-eSign-UserIcon2.png)

  
Bug Fixes

1\. GPES-4670  
We fixed an issue with the Bulk Send option being removed from the left navigation.   
Previously, when an enterprise account viewed a completed bulk sessions the feature would disappear from the left navigation.  
Now, this issue has been resolved. 

2\. GPES-4864  
We’ve fixed an issue with signatures not displaying on some archived signing session documents.   
Previously, in some cases the signature was not displaying on some archived documents even though the signer had completed the signing session, and the certificate confirms the signer did in fact sign the document  
Now, this issue has been resolved.

3\. GPES-4862  
We fixed an issue with documents from a completed session not being downloaded to a mobile device.  
Previously, when a signer clicked the link to download documents in the session complete email message, the documents were not downloading to their mobile device. This issue was specific to the mobile Chrome browser.  
Now, this issue has been resolved.
